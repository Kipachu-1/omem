# example-project

Memory for one project — architecture decisions, conventions, gotchas, deploy config.
Rename this folder to your project's name (and update `island:` in notes that land here).

**Who writes here:** agents working on this project, and you.

**Conventions:**
- Tag prefix: `example-project/` (e.g. `example-project/api`, `example-project/gotcha`).
- Pin decisions that affect the whole project (`pinned: true`).
- Note deprecations and migration paths explicitly.

**Common notes:** build commands, API contracts, schema decisions, recurring bugs and their fixes.
